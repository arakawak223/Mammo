## 1. プロジェクト背景と目的

### 1.1 背景

- 特殊詐欺の被害額は年間400億円超で過去最悪水準を更新
- 被害者の約80%が65歳以上の高齢者
- 暴力団組織が「闇バイト」を使った強盗や詐欺の実行役を大量動員
- 既存の防犯アプリ（デジポリス等）は「高齢者本人が操作する」前提であり、ITリテラシーの壁で実効性が低い

### 1.2 目的

高齢者の操作負担をゼロに近づけ、家族がリモートで監視・対策を実行できる防犯アプリを開発する。

### 1.3 コアコンセプト

1. **高齢者のスマホ = 詐欺検知センサー**：操作を強いない。バックグラウンドで自動検知。
2. **家族 = リモート管理者**：通知を受け、分析し、遠隔で対策を打つ。
3. **AIが橋渡し**：音声解析・スコアリングで判断を支援。

### 1.4 プラットフォーム戦略

iOSでは通話ログ取得・SMS本文転送・バックグラウンド通話制御などに厳しい制限がある。この制約を**「だからこそ、このアプリが必要である」**というストーリーに転換する。

| プラットフォーム | 位置づけ | 提供機能 |
|----------------|---------|----------|
| **Android（高齢者側）** | フル機能版センサーアプリ | F1〜F9 すべて |
| **iOS（高齢者側）** | 制限付きセンサーアプリ | F1, F9, F2(一部), F3(通知タップ起動) |
| **iOS / Android（家族側）** | フル機能管理アプリ | ダッシュボード、リモート操作すべて |

**戦略的根拠**: 「親がAndroid、子がiPhone」という構成は日本の世帯で非常に多い。高齢者にはAndroid端末（らくらくスマホ等）が普及しており、家族（30〜50代）はiPhoneが多い。この組み合わせをターゲットの主軸に据え、**Android版をフル機能版として先行リリース**し、iOS版は家族側の管理アプリとして同時リリースする。

## 2. ターゲットユーザーと役割

### ユーザーA（高齢者 / 被保護者）

| 項目 | 内容 |
|------|------|
| 年齢層 | 65歳以上 |
| ITスキル | 低〜なし（電話・LINEが精一杯） |
| 操作期待値 | ほぼゼロ（初期設定は家族が行う） |
| スマホ上の役割 | センサー端末（自動検知＋1タップ通報） |

### ユーザーB（家族 / 管理者）

| 項目 | 内容 |
|------|------|
| 年齢層 | 30〜60代 |
| ITスキル | 中〜高（スマホアプリの通常操作が可能） |
| 操作期待値 | ダッシュボードで通知確認・分析・遠隔操作 |
| スマホ上の役割 | 管理コンソール |

### ペアリング関係

- 1人の高齢者に対して複数の家族（最大5名）を紐づけ可能
- 1人の家族が複数の高齢者を管理可能（M:N 関係）
- 管理者権限レベル：オーナー（1名）/ メンバー（複数名）

## 3. 機能一覧（サマリー）

| # | 機能名 | 側 | MVP | 概要 |
|---|--------|-----|-----|------|
| F1 | 「これ詐欺？」ボタン | 高齢者 | Phase 1 | 1タップで家族に不審状況を転送 |
| F2 | 自動転送機能 | 高齢者 | Phase 1 | 不審な着信・SMSを家族へ自動通知 |
| F3 | AI音声アシスタント | 高齢者 | Phase 2 | 通話後に自動発話し音声のみで家族相談を完了 |
| F4 | リアルタイム・アラート | 家族 | Phase 1 | 不審挙動を即座にプッシュ通知 |
| F5 | 会話サマリー（AI解析） | 家族 | Phase 2 | 音声テキスト化＋危険度スコアリング |
| F6 | リモート・ブロック | 家族 | Phase 2 | 遠隔で着信拒否リストを編集 |
| F7 | 被害額データ連動 | 家族 | Phase 3 | 地域別の流行手口と対策アドバイス |
| F8 | 闇バイトチェッカー | 共通 | Phase 3 | SNS求人文の組織犯罪リスク判定 |
| F9 | 緊急ロック（緊急SOS） | 共通 | Phase 1 | 位置情報＋周囲音声を家族へ強制送信 |
